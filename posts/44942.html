利用hexo与Git建立个人博客之错误总结

一、node.Js下载

二、Git注册与Github Pages创建

三、Git用户名和邮箱注册

前面几点（包括后面没有详细讲诉的内容）直接参考大佬博客：https://yafine-blog.cn/posts/4ab2.html

四、本地安装hexo静态博客框架以及发布到Github Pages

1、建立一个博客文档-blogtest

2、右键进入Git Bash Here 分别输入一下指令

  npm install -g hexo-cli   

hexo init 文件名

   cd 文件名

   Npm install

①在输入时如果遇到：NpmERR!path

E:\Node.js\node_global\node_modules\hexo\bin\hexo的情况

解决方法：

直接找到C:\Users\Administrator中的npmrc文件，将其删除

②无法找到这个指令

解决方法：

1、 检查版本，使用Git bash 输入node -v npm -v

2、 检查一下自己的path环境变量是否存在问题

​ )

发现是在用户的path所用的地址出错了，将其改成图片上的node_global的地址。

运行结束后，在接下来输入hexo g和hexo s，会马上显示出

其中 http://localhost:4000 是你本地的博客

3、本地博客发表到Github Pages

基本的博客建设差不多就是这样子了，接下来就是博客的优化处理

①在优化主题时，主题无法启动？

在企图使用git指令启动本地博客时，出现了报错，说是找不到最近的文件

解决办法：

在子目录里使用git指令，例如我一开始是使用F:\blogtest，出现报错

改成F:\blogtest\blogtest在这个目录下使用git即可，并输入

​ npm install hexo-server –save

​ hexo s->启动本地博客

②但是这样子只是能够访问电脑上的本地地址http://localhost:4000/，在其他电脑就不可以访问，这个怎么处理？（如果我更换了主题文件里_config.yml的内容，然后怎么处理？）

在这个时候，就需要在含有package.json的文件，在我这里就是F:\blogtest\blogtes，分别输入

hexo clean

hexo g

hexo s

hexo d

这四条指令，可以将自己的主题传送到自己的Github上，就可以实现在自己的域网上登录了

注意：如果没改成功，可能是因为与浏览器的缓存有关

③更换打赏二维码一直没成功

可能出现的原因是：

照片规格不规范，建议是大概以二维码的实际大小进行裁剪

④在输入hexo g出现Theme config load failed Process\failed

将主题的_config.yml的每一个项目后面添加一个空格

⑤hexo的主页出现404，无法进入，而且在输入hexo g显示

说明缺少了wordcount插件

需要添加插件

npm i –save hexo-wordcount

⑥在改变subtitle的值时，重著hexo时出现使用hexo g,报错——>想改变文章中间的值

改错地方了，应该在hexo根目录下的配置文件更改

⑦相册的导入失败

——没有在Github建立相册，只是在本地文件力建立了，在hexo中无法识别（未解决）

解决办法：

建立一个库，将本地照片上传到Github既可

遇到的阻碍：

7.1 在新建一个文件，输入git init，报错:FATAL E:\blog not empty, please run hexo init on an empty folder and then copy your files into it。

没有在根目录下建立，而是在myblog里面建立，应该在这个目录下建立

一开始，我是在这个目录下下载的，所以一直报错

4、发表博客文章篇

发表文章一共分为有两个部分，一部分是文字，一部分就是图像，现在我开始讲诉一下，怎么处理这两个部分

①发表文字（最基础）

1.1 在自己博客的根目录下，使用git bash指令：

new hexo 文章名字

指令输入完后，在F:\blogtest\blogtest\source_posts这里，是存放着你的hexo文章的地方，在这里，你可以看到后缀为.md的文件，

1.2 点击你新建文章，开始编写

如果是用本地管理员权限来编写的话，是十分繁琐的，所以我建议是使用编译器进行编写，推荐是用Typora0.860.9.86（最新版）编译器

下载地址： https://typora.io/

1.3 编写完之后就可以将其上传到Github，使用

Hexo clean

Hexo g

Hexo s ->查看hexo是否有问题

Hexo d

②上传照片（详细解说）

如果你开始是直接将在其他地方编写的文章粘贴到你自己的博客里的话，会出现这样的情况：

据你所视，照片是无法上传到你自己的博客的

2.1 原因：

I 这个主要的问题是因为电脑是无法识别你自己本地里的照片的，

II 所以需要将照片上传到一个网址上，这样子就必定要将一个专门存放照片的文件夹，

III 然后将你需要上传的照片一张张存放在这个文件里，然后将其上传到一个网站上，如Github

IV最后在发布博客时，把照片的地址copy下去就可以了。

2.2一开始我的解决方法是：

I 在github新建一个库，专门来存放照片的

II 在根目录下，新建一个文件夹，我的设置为Myphono，

III 使用git指令，将其连接自己的Github上

但是！这个方法十分繁琐，不利于大量照片的上传。

所以要那种可以直接上传照片，电脑又可以识别的编译器，例如时CDSN，简书的编译器

故我发现了Picgo2.3.0-Bate 3，github，typora0.9.86三者可以实现图片全自动上传

③Picgo2.3.0-Bate 3，github，typora三者可以实现图片全自动上传
